# Notes on ST3020 Servo

We use WaveShare ST3020 servos in the v1 version of VESICOLOS. Here
are some notes pulled together from the various documentation sources.

The servo is controlled by writing numbers such as the target position
or target speed directly into EEPROM and SRAM registers. Values are
either bytes, or big-endian words with most-significant bit as a sign bit
in the case of signed words. (The latter deviates from what driver codes
seem to assume.)

Below, the `SMS_STS` codes are used by the WaveShare demo code, in the
case of words they get appended `_L` and `_H` for low and high byte,
with the high byte being an address of one more (that's what big-endian means).


## EEPROM registers

| address (hex) | address (dec) | datatype | name                        | function                  |
|---------------|---------------|----------|-----------------------------|---------------------------|
| 0x00          | 0             | byte     |                             | firmware major version    |
| 0x01          | 1             | byte     |                             | firmware minor version    |
| 0x03          | 3             | byte     | `SMS_STS_MODEL_L`           | servo main version        |
| 0x04          | 4             | byte     | `SMS_STS_MODEL_H`           | servo version             |
| 0x05          | 5             | byte     | `SMS_STS_ID`                | servo id 0..253           |
| 0x06          | 6             | byte     | `SMS_STS_BAUD_RATE`         | baud rate (see below)     |
| 0x07          | 7             | byte     |                             | delay in 2µs units        |
| 0x08          | 8             | byte     |                             | 1: return packet on write |
| 0x09          | 9             | uword    | `SMS_STS_MIN_ANGLE_LIMIT_L` | min angle limit           |
| 0x0B          | 11            | uword    | `SMS_STS_MAX_ANGLE_LIMIT_L` | max angle limit           |
| 0x0D          | 13            | byte     |                             | max temperature limit     |
| 0x0E          | 14            | byte     |                             | max input voltage         |
| 0x0F          | 15            | byte     |                             | min input voltage         |
| 0x10          | 16            | uword    |                             | max torque                |
| ...           | ...           | ...      |                             | stuff for PWM mode        |
| 0x21          | 33            | byte     | `SMS_STS_MODE`              | operating mode            |

Baud rates are encoded as

| value of 0x06 register | baud rate |
|------------------------|-----------|
| 0                      | 1000000   |
| 1                      | 500000    |
| 2                      | 250000    |
| 3                      | 115200    |
| 4                      | 57600     |
| 5                      | 38400     |
| 6                      | 19200     |
| 7                      | 9600      |

Operating modes are

| value of 0x21 register | operating mode               |
|------------------------|------------------------------|
| 0                      | servo mode, position control |
| 1                      | wheel mode, velocity control |
| 2                      | PWM open loop                |
| 3                      | stepper mode                 |

## SRAM registers

| address (hex) | address (dec) | datatype | name                          | function                  |
|---------------|---------------|----------|-------------------------------|---------------------------|
| 0x28          | 40            | byte     | `SMS_STS_TORQUE_ENABLE`       | torque switch             |
| 0x29          | 41            | byte     | `SMS_STS_ACC`                 | acceleration              |
| 0x2A          | 42            | sword    | `SMS_STS_GOAL_POSITION_L`     | target position           |
| 0x2C          | 44            | word     | `SMS_STS_GOAL_TIME_L`         | bit 10 = direction, PWM?  |
| 0x2E          | 46            | sword    | `SMS_STS_GOAL_SPEED_L`        | target velocity           |
| 0x30          | 48            | word     | `SMS_STS_TORQUE_LIMIT_L`      | torque limit              |
| 0x37          | 55            | byte     | `SMS_STS_LOCK`                | to lock EEPROM            |
| 0x38          | 56            | sword ro | `SMS_STS_PRESENT_POSITION_L`  | current position          |
| 0x3A          | 58            | sword ro | `SMS_STS_PRESENT_SPEED_L`     | current velocity          |
| 0x3C          | 60            | uword ro | `SMS_STS_PRESENT_LOAD_L`      | current torque            |
| 0x3E          | 62            | byte ro  | `SMS_STS_PRESENT_VOLTAGE`     | current voltage           |
| 0x3F          | 63            | byte ro  | `SMS_STS_PRESENT_TEMPERATURE` | current temperature       |
| 0x40          | 64            | byte ro  |                               | async write flag?         |
| 0x41          | 65            | byte ro  |                               | servo status              |
| 0x42          | 66            | byte ro  | `SMS_STS_MOVING`              | 1 if servo is moving      |
| 0x45          | 69            | word ro  | `SMS_STS_PRESENT_CURRENT`     | current current           |

The "torque switch" value can be set to 128 in which case the position is reset to the middle (2048).


## Summary

Typical operations we need for VESCICOLOS:

| action     | address | value  |
| wheel mode | 0x21    | 1      |
| servo mode | 0x21    | 0      |
| set middle | 0x28    | 128    |
| is moving? | 0x42    | /read/ |
